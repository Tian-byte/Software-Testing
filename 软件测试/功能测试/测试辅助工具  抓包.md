# 测试辅助工具

- 能够做抓包工具抓取 http 请求和响应的数据包
- 能操作抓包工具进行弱网测试
- 能使用抓包工具绕过页面限制进行测试

### http 协议

面试题：

http和https的区别

- http 不加密，不安全，https 加密 安全  https 在公网使用
- http 使用的是80端口  https 使用的是442端口
- 两者都是协议，都可以用来传输协议



请求方法：

get 请求

- get 请求参数放在url中 在浏览器地址栏可以看到
- url本身的长度受限制（浏览器的长度有限制）
- 例如搜索

​	

post 请求

- 请求参数放在请求体里面（通过抓包可以看见）--- 相对安全
- 消息本身没有长度限制，因此post请求可以传递任意长度的参数
- 例如登录



请求行： 请求方法 url 协议/协议版本号  请求方发  url 

请求体： 组成： 类似键值对的格式   描述客户端的信息

请求体：name = value & xxx = xxx





响应行： 协议的版本号，响应状态码，状态的描述

响应头：响应服务器的属性信息

响应体： 服务器返回的结果（图片/html/json/txt）



响应状态码：2xx 成功	3xx 重定向  4xx 客户端  5xx  服务器



### 抓包

说明：客户端向服务器发送请求以及服务器响应客户端的请求，都是以数据包来传递的



转包：（packet capture) 通过工具拦截客户端与服务器交互的数据包



抓包工具（fiddler) 

1. fiddler 是一个http 协议调试代理工具，他能够记录 并检查所有你的电脑和互联网的http通讯



常见的三种应用场景

1. 定位前后端的bug
2. 弱网测试
3. 绕过界面限制直接测试服务器



如何定位Bug 是前端的问题还是后端的问题？

- 如果抓不到包 显然是前端的问题
- 如果有请求但是没有响应 那就是后端的问题
- 如果有请求也有响应，需要查看相应信息，如果返回报错，则需要具体分析报错的内容



​      

### 弱网环境下可能会出现的异常

- 上传文件时进度卡动
- 登录不上或者登录后立即掉线
- 响应过程中页面空间可以点击，导致崩溃
- 搜索不响应点击后显示总在刷新被替换



#  测试报告的书写

### 测试报告的主要内容

- 测试项目概述
- 测试过程回顾
- 测试统计分析
- 测试结果确认
- 测试工作总结与改进
